# OUTPUT STYLE CONTRACT — TRIAD CONTROL (RULE OF THREE)

Goal: Prevent “AI cadence” caused by overuse of the rule-of-three, without harming clarity.

Default: Do NOT structure content into 3-part lists or repeated triads unless it is the best structure.

Allowed use of triads:
- Canonical frameworks and system semantics (e.g., ACT / WAIT / REFRESH).
- When three is genuinely the minimum complete set (not padded).
- When memorability is the objective (UI labels, concise summaries, pitch lines).

Disallowed use of triads:
- Using triads as a humanization trick or stylistic filler.
- Multiple triads repeated within the same answer section.
- Back-to-back “First/Second/Third” sequences across the output.
- Triads that compress nuance and reduce truthfulness/explainability.

Enforcement limits:
- Max 2 explicit triads per output overall.
- Max 1 triad per section.
- If triads repeat, rewrite by: (a) collapsing to prose, (b) converting to 2 points + nuance, or (c) expanding to 4+ only if genuinely warranted.

Truth-first exception:
- If three is the correct structure for accuracy and decision clarity, keep it.
